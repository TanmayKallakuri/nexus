7. KNOWN RISK / REVIEW POINTS
Some scored cognitive features depend on hard-coded answer keys.
These include CRT, numeracy, financial literacy, vocabulary, and Wason-style items.
These should be sanity-checked on a sample before the team treats them as fully trusted signal.

8. WHAT THIS DOES NOT DO
- Does not predict hidden test answers
- Does not perform clustering
- Does not perform question retrieval
- Does not use an LLM
- Does not replace later ensemble models

9. HOW TEAMMATES SHOULD USE IT
Downstream models should read outputs/person_response_profiles.csv as the main person-level feature matrix.
Use the data dictionary to understand feature meanings before feature selection or modeling.
